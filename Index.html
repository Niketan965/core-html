Ordered List of Html tags

  1. H1-H6 - heading tags
  2. DOCTYPE! - This declaration defines the document type and version of HTML being used.
  3. html : The root element that contains all other HTML elements on the page.
  4. head: Contains meta-information about the document, such as the title, character set,
  5. title: Sets the title of the web page, which appears in the browser's title bar or tab.
  6. meta: Provides metadata about the document, such as character encoding, authorship, and description for search engines.
  7. link>: Links external resources like stylesheets to the HTML document.
  8. style>: Used to define inline CSS styles for a specific section of the document.
  9. script>: Embeds or links to JavaScript code for enhancing interactivity and functionality.
  10. body>: Contains the main content of the web page, including text, images, and other elements that are visible to the user.
  11. header>: Typically contains introductory content or navigation elements for the top of a web page.
  12. nav>: Represents a section of navigation links, often found in headers or sidebars.
  13. main>: Represents the primary content of the document and should be unique within the page.
  14. article>: Defines a self-contained piece of content, such as a blog post, news article, or forum post.
  15. section>: Groups related content together and provides a way to apply CSS styles and JavaScript behaviors to them.
  16. aside>: Contains content that is tangentially related to the surrounding content, often used for sidebars.
  17. footer>: Typically found at the bottom of a web page, containing information like copyright notices and contact details.
  18. h1, h2>, h3>, h4>, h5>, h6>: Headings, which define the hierarchy of content on the page, with h1 being the highest and h6> the lowest.
  19. p>: Represents a paragraph of text.
  20. a>: Creates hyperlinks to other web pages or resources.
  21. img>: Embeds images on the page.
  22. ul>: Creates an unordered (bulleted) list.
  23. ol>: Creates an ordered (numbered) list.
  24. li>: Defines list items within ul or ol lists.
  25. table>: Defines a table.
  26. tr>: Defines a table row.
  27. td>: Defines a table cell in a row.
  28. form>: Contains input elements and defines a web form for user input.
  29. input>: Represents an input field within a form.
  30. textarea>: Defines a multi-line text input field within a form.
  31. button>: Creates a clickable button within a form or elsewhere on the page.

Q5

A full-stack web development tech stack typically includes a combination of technologies and tools for both front-end and back-end development. Here's a general overview of a common full-stack web development tech stack:

Front-End Technologies:

HTML/CSS
These are the foundational languages for creating the structure and styling of web pages.
JavaScript
The primary language for adding interactivity and functionality to web applications. Popular libraries and frameworks include React, Angular, or Vue.js.
Version Control
Tools like Git for source code management and collaboration.

Back-End Technologies

Server-Side Programming:
You can choose from various programming languages for the back end, such as Node.js (JavaScript), Python, Ruby, Java, or PHP.
Web Frameworks:
Frameworks like Express.js (Node.js), Django (Python), Ruby on Rails (Ruby), Spring Boot (Java), or Laravel (PHP) provide a structured way to build server-side logic.
Database:
Choose a database system like MySQL, PostgreSQL, MongoDB (NoSQL), or others, depending on your project's requirements.
APIs
Develop RESTful or GraphQL APIs to facilitate communication between the front end and back end.
Authentication and Authorization:
Implement user authentication and authorization systems using tools like Passport.js or OAuth.
Web Servers:
Deploy your application on web servers like Nginx or Apache.
